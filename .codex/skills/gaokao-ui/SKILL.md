# gaokao UI

先看页面结构，再改样式。

UI 问题通常是信息架构、组件语义、状态流和布局约束的问题。

## 必读上下文

修改 UI 前先读取：

- 目标页面。
- 目标组件。
- 路由配置。
- API client。
- 状态管理。
- 全局样式。
- `spec.md` 中的当前产品事实。

## 信息架构

- 管理端服务高频维护：列表、筛选、批量操作、表单、详情、审核和结果查看。
- 考试端服务稳定作答：考试信息、倒计时、题目区、答题卡、保存状态、提交确认和结果反馈。
- 题库、试题、试卷、考试、成绩必须有清晰路径。
- 列表页负责检索、筛选和进入编辑。
- 编辑页负责字段填写、校验和保存。
- 详情页负责只读查看和审计信息。

## 组件语义

- 按钮只用于真实命令。
- 表格用于可比较数据。
- 表单用于编辑。
- Dialog 用于确认和短流程。
- Drawer 用于辅助详情。
- Tabs 用于同一对象的不同视图。
- Steps 用于有顺序的考试发布或组卷流程。

## 布局规则

- 管理端保持密度适中、可扫描、可批量操作。
- 考试端保持稳定、克制、无干扰。
- 长文本必须换行或省略。
- 表格列必须明确宽度、固定列和溢出策略。
- 表单项必须有清楚 label、校验提示和错误状态。
- 页面不使用装饰性大面积渐变和空洞宣传卡片。

## 验证

收尾前检查：

- 文本不溢出。
- 按钮不被挤压。
- 表单错误提示可见。
- 空状态、加载态、失败态存在。
- 移动宽度下考试端仍可作答。
