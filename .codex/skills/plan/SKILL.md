# Plan

`plan.md` 是 kaoshi 当前任务的单文件规格驱动执行合同。

## 写作内容

`plan.md` 必须覆盖：

- 需求。
- 当前事实。
- 生产级验收保护点。
- 目标。
- 不做范围。
- 设计。
- 实施任务。
- 验证计划。
- 收口。

## 执行规则

- 写完计划，再动手。
- checklist 必须随进度更新。
- 新事实推翻计划，先改计划，再继续。
- 不用口头解释替代计划更新。
- 不偷偷偏离计划。
- research、设计、实现、测试、文档同步、验证和收口必须形成完整交付。
- 分阶段只表示交付边界，不表示降低标准。
- 当期范围必须按生产级标准完成。

## 快速演进规则

kaoshi 是快速演进项目。计划必须服务当前生产级主干。

- 当前事实高于历史形态。
- 需要重建时直接重建当前主干。
- 数据结构变化以当前初始化脚本、清空数据库和重新验收为准。
- 历史数据保留、旧接口保留和旧页面保留只在 owner 明确要求并写入当次 `plan.md` 后执行。

## 验证收口

收口必须写清：

- 完成了哪些当前事实。
- 运行了哪些命令。
- 哪些测试通过。
- 是否存在剩余风险。

生产级任务优先使用：

```powershell
python .\start_test.py
python .\start_browser_test.py
```
