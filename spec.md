# kaoshi Spec

`spec.md` 是 kaoshi 的当前产品事实主干。它记录已经确认的产品定位、业务边界、核心概念、用户路径、技术方向和验收标准。

## 1. 产品定位

kaoshi 是一个现代化考试与考试管理平台。

平台服务于学校、机构、团队和个人组织考试的完整流程：建设题库、维护试题、保存考试草稿、发布考试、考生作答、自动评分、成绩归档和结果查看。考试发布快照和作答快照承载题目、选项、答案、分值和附件的历史事实，不作为管理端显性同级入口。

## 2. 当前技术方向

- 后端使用 Java 21 和 Spring Boot 3。
- 数据库使用 MySQL 8。
- ORM 使用 MyBatis-Plus。
- 认证权限使用 Spring Security 和 JWT。
- 缓存、限流和考试运行状态辅助能力使用 Redis。
- 管理端使用 Vue 3、TypeScript、Vite 和 Element Plus。
- 在线考试端使用 Vue 3、TypeScript、Vue Router 和 Element Plus，并集成在统一控制台中。
- 部署使用 Docker Compose 组织 MySQL、Redis、后端服务和前端服务。
- 当前快速演进期默认清空重建本地数据；数据库脚本表达当前初始化事实，不做旧库迁移兼容。

## 3. 核心用户

- 系统管理员：管理平台配置、用户、角色、权限和全局数据。
- 考务管理员：维护题库、试题、试卷、考试安排和成绩。
- 教师或命题人：创建题目、维护解析、组卷和查看考试结果。
- 考生：查看考试任务、进入考试、作答、提交和查看结果。

## 4. 核心概念

### 4.1 用户与权限

用户是平台身份主体。用户可归属部门。角色和权限控制管理端能力、考试发布能力、题库维护能力和成绩查看范围。

当前已实现管理员登录、当前用户查询、首次登录强制改密、用户分页搜索、用户新建、用户编辑、用户部门归属、用户启用禁用、用户角色分配、用户 Excel 模板下载、用户 Excel 导入、用户 Excel 导出、角色新建、角色编辑、角色权限分配和角色菜单分配。用户列表不展示导入时间或创建时间。用户导入模板的样例和字典清单来自当前有效部门、角色和状态；新建和导入用户默认密码为 `123456`，后端保存哈希，登录后必须修改密码。

当前系统管理显性入口只保留角色管理、部门管理和用户管理，顺序也是角色、部门、用户。权限与菜单数据保留为角色配置的数据来源和后端鉴权契约，不作为普通用户显性页面入口。

### 4.1.1 部门

部门用于组织用户和考务管理边界。部门支持无限层级树、部门编码、名称、说明和状态。

当前已实现左侧部门树、右侧当前部门详情、直属下级列表、在线新建、在线新建下级、编辑、删除、删除有下级部门的冲突保护和部门编码自动生成。部门不提供 Excel 导入导出。

### 4.2 题库

题库是试题的组织容器。题库支持分类、标签、题型、难度、知识点和归属范围。

当前已实现试题分类、题库树、新建分类、编辑分类、删除空分类、新建题库、编辑题库、题库状态维护和题库题量统计。试题分类是一层结构，分类下挂题库，题库下挂试题；已有题库的分类不能删除。题库节点展示总题量、单选题量和多选题量。管理端只保留“题库管理”显性入口承载题库和试题：左侧题库树，右侧当前题库试题列表。

### 4.3 试题

试题是考试内容的基本单位。试题包含题干、题型、选项、答案、解析、难度、标签和附件。试题自身不保存固定分值，分值属于考试发布快照中的题目事实。

当前已实现单选题、多选题、选项、正确答案、解析、难度、状态和附件。附件支持上传真实图片、音频、视频、文件，也支持录入外部或本地 URL，并支持在试题编辑页上移下移调整展示顺序。初始化样例包含本地音频、png 图片和 jpg 图片附件。

当前已实现试题按题库树筛选、新建、编辑、Excel 模板下载、Excel 导入和 Excel 导出。导入字段覆盖题库名称、题型、难度、题干、选项、正确答案和解析。必填字段为题库名称、题型、题干、选项A、正确答案；正确答案使用 `A`、`B`、`AC` 这类连续字母格式，不使用逗号。Excel 导入只处理文本结构化题目，富媒体附件在网页试题编辑页上传或填写 URL。Excel 模板生成、导入和导出使用 Apache POI。试题新建、编辑和导入后会重新读取题库题量和试题列表。

### 4.4 发布快照

发布快照由考试发布动作生成。发布快照记录发布时的题目、题型、题干、解析、分值、题目顺序、选项、正确答案和附件，后续题库或试题编辑不会改写已生成的发布快照。

当前已实现按考试草稿题目明细生成发布快照。考试草稿题目明细可由组卷规则生成，也可从试题列表手工加入，支持保存读回、人工排序、修改分值、移除题目和发布前预览；草稿题目明细保存题干、解析、选项、正确答案和附件快照，后续题库或试题编辑不会隐式改写草稿试卷。管理员点击显式“更新试卷”后，才会按当前题库和试题重新生成草稿题目明细。发布时按草稿题目明细的题目、分值、顺序、选项、正确答案和附件生成发布快照。发布后，后续题库或试题编辑不会改写已生成的发布快照。

### 4.5 考试

考试保存组卷规则并定义可见范围、开始时间、结束时间、限时、及格分、提交规则、成绩规则和发布状态。

当前已实现考试列表、新建考试、编辑考试、保存草稿、发布考试、关闭考试、复制试卷、下载试卷、撤销发布、按考试查看成绩、按题库规则组卷、从试题列表手工选题、单选/多选数量与分值配置、题目明细生成、显式更新试卷、题目明细排序、题目明细预览、总分自动计算、题量自动计算、及格分、考试时长、是否限时、可考次数、题目显示方式、题目顺序、完全公开和部门开放。复制试卷会复制考试配置、开放范围、组卷规则和题目明细并生成新草稿。下载试卷会导出当前发布快照或草稿题目明细。撤销发布只允许没有作答记录和成绩记录的已发布考试回到草稿；已有作答记录后不允许撤销发布。可考次数支持无限次和限定次数；题目显示方式支持逐题显示和整卷一页；题目顺序支持固定顺序和随机顺序；部门开放考试会保存开放部门，并按用户部门过滤考试任务和开始考试权限。

### 4.6 作答

作答记录考生每道题的答案、用时、提交状态、评分结果和最终成绩。

当前已实现考试中心、准备考试、开始作答、attempt 快照、按考试配置逐题作答或整卷一页作答、固定/随机顺序在 attempt 内固化、上一题、下一题、按题型分组答题卡、题目正文显示单选题/多选题和分值、题目图片/音频/文件附件展示、作答选择后保存到后端、刷新或重新进入后读回已保存答案、答案提交、提交锁定、重复提交拒绝和服务端考试时长校验。考生作答页不提供逐题/整卷临场切换。

### 4.7 成绩

成绩是作答提交后的结果归档，包含总分、得分、正确率、用时、题目明细和可查看范围。

当前已实现基于 attempt 快照的选择题自动评分、总分、得分、正确题数、是否通过、提交时间、考生个人成绩列表、管理员按考试查看考生成绩列表、成绩详情、题目明细、考生答案、正确答案、解析和附件展示。初始化数据库包含多个考生和 10 条已提交成绩样本，便于人工检查成绩管理。

## 6. 验收标准

- 根目录存在 `README.md`、`AGENTS.md`、`spec.md`、`plan.example.md`、`LICENSE`、`SECURITY.md`、`CONTRIBUTING.md` 和 `.gitignore`。
- `.codex/skills/` 中存在开发、UI 和 plan 三个项目级 skill。
- 文档统一使用 kaoshi 作为项目名。
- 文档只记录当前确认的产品方向和工程规则。
- `backend/` 存在可测试的 Spring Boot 工程。
- `frontend/` 存在可类型检查和可构建的 Vue 3 TypeScript 工程。
- 管理员可以通过管理端维护角色、权限关系、菜单关系、部门和用户，并通过 Excel 导入导出用户。
- 管理员可以通过管理端在线维护部门树，并清楚查看当前部门、上级部门和直属下级。
- 管理员可以通过管理端维护题库、题库题量、试题和考试，保存考试草稿，按规则或手工选题形成题目明细，显式更新试卷，预览、排序和下载试卷，并在发布时按题目明细生成发布快照。
- 管理员可以通过 Excel 模板导入和导出试题。
- 登录用户可以通过在线考试查看考试任务、准备考试、开始作答、保存答案、刷新读回答案、提交试卷并查看成绩详情。
- 在线考试可以展示题目图片、音频和文件链接附件。
- 系统可以基于作答快照对单选题和多选题自动评分，归档成绩，并支持管理员按考试查看考生成绩。
- 未登录请求不能访问管理接口。
- `start_test.py` 是当前常规验证入口，不打开浏览器。
- `start_browser_test.py` 是当前真实浏览器验收入口，会重置本项目 Docker 数据库并运行有头 Chromium 自动验收。
- `stop_test.py` 可以停止测试相关服务。
