# kaoshi Spec

`spec.md` 是 kaoshi 的当前产品事实主干。它记录已经确认的产品定位、业务边界、核心概念、用户路径、技术方向和验收标准。

## 1. 产品定位

kaoshi 是一个现代化考试与考试管理平台。

平台服务于学校、机构、团队和个人组织考试的完整流程：建设题库、维护试题、保存考试草稿、发布考试、考生作答、自动评分、成绩归档和结果查看。考试发布快照和作答快照承载题目、选项、答案、分值和附件的历史事实，不作为管理端显性同级入口。

## 2. 当前技术方向

- 后端使用 Java 21 和 Spring Boot 3。
- 数据库使用 MySQL 8。
- ORM 使用 MyBatis-Plus。
- 认证权限使用 Spring Security 和 JWT。
- 缓存、限流和考试运行状态辅助能力使用 Redis。
- 管理端使用 Vue 3、TypeScript、Vite 和 Element Plus。
- 在线考试端使用 Vue 3、TypeScript、Vue Router 和 Element Plus，并集成在统一控制台中。
- 部署使用 Docker Compose 组织 MySQL、Redis、后端服务和前端服务。
- 当前快速演进期默认清空重建本地数据；数据库脚本表达当前初始化事实，不做旧库迁移兼容。

## 3. 核心用户

- 系统管理员：管理平台配置、用户、角色、权限和全局数据。
- 考务管理员：维护题库、试题、试卷、考试安排和成绩。
- 教师或命题人：创建题目、维护解析、组卷和查看考试结果。
- 考生：查看考试任务、进入考试、作答、提交和查看结果。

## 4. 核心概念

### 4.1 用户与权限

用户是平台身份主体。用户可归属部门。角色和权限控制管理端能力、考试发布能力、题库维护能力和成绩查看范围。

当前已实现管理员登录、当前用户查询、首次登录强制改密、用户分页搜索、用户新建、用户编辑、用户部门归属、用户启用禁用、用户角色分配、用户 Excel 模板下载、用户 Excel 导入、用户 Excel 导出、角色新建、角色编辑、角色权限分配和角色菜单分配。新建和导入用户默认密码为 `123456`，后端保存哈希，登录后必须修改密码。

当前系统管理显性入口只保留用户管理、角色管理和部门管理。权限与菜单数据保留为角色配置的数据来源和后端鉴权契约，不作为普通用户显性页面入口。

### 4.1.1 部门

部门用于组织用户和考务管理边界。部门支持无限层级树、部门编码、名称、说明和状态。

当前已实现部门树、在线新建、编辑、删除、删除有下级部门的冲突保护和部门编码自动生成。部门不提供 Excel 导入导出。

### 4.2 题库

题库是试题的组织容器。题库支持分类、标签、题型、难度、知识点和归属范围。

当前已实现试题分类、题库列表、新建题库、编辑题库和题库状态维护。试题管理可按题库筛选，列表展示题库归属、题型和正确答案。

### 4.3 试题

试题是考试内容的基本单位。试题包含题干、题型、选项、答案、解析、难度、标签和附件。试题自身不保存固定分值，分值属于考试发布快照中的题目事实。

当前已实现单选题、多选题、选项、正确答案、解析、难度、状态和附件。附件支持上传真实图片、音频、视频、文件，也支持录入外部或本地 URL。初始化样例包含本地音频、png 图片和 jpg 图片附件。

当前已实现试题 Excel 模板下载、试题 Excel 导入和试题 Excel 导出，导入字段覆盖题库名称、题型、难度、题干、选项、正确答案和解析。必填字段为题库名称、题型、题干、选项A、正确答案；正确答案使用 `A`、`B`、`AC` 这类连续字母格式，不使用逗号。Excel 导入只处理文本结构化题目，富媒体附件在网页试题编辑页上传或填写 URL。Excel 模板生成、导入和导出使用 Apache POI。

### 4.4 发布快照

发布快照由考试发布动作生成。发布快照记录发布时的题目、题型、题干、解析、分值、题目顺序、选项、正确答案和附件，后续题库或试题编辑不会改写已生成的发布快照。

当前已实现按题库规则生成发布快照，固定基础顺序按规则顺序、题型顺序和题目 ID 顺序生成。

### 4.5 考试

考试保存组卷规则并定义可见范围、开始时间、结束时间、限时、及格分、提交规则、成绩规则和发布状态。

当前已实现考试列表、新建考试、编辑考试、保存草稿、发布考试、关闭考试、按题库规则组卷、单选/多选数量与分值配置、总分自动计算、题量自动计算、及格分、考试时长、是否限时、可考次数、题目显示方式、题目顺序、完全公开和部门开放。可考次数支持无限次和限定次数；题目显示方式支持逐题显示和整卷一页；题目顺序支持固定顺序和随机顺序；部门开放考试会保存开放部门，并按用户部门过滤考试任务和开始考试权限。

### 4.6 作答

作答记录考生每道题的答案、用时、提交状态、评分结果和最终成绩。

当前已实现考试中心、准备考试、开始作答、attempt 快照、按考试配置逐题作答或整卷一页作答、固定/随机顺序在 attempt 内固化、上一题、下一题、按题型分组答题卡、题目正文显示单选题/多选题和分值、题目图片/音频/文件附件展示、答案提交、提交锁定和重复提交拒绝。考生作答页不提供逐题/整卷临场切换。

### 4.7 成绩

成绩是作答提交后的结果归档，包含总分、得分、正确率、用时、题目明细和可查看范围。

当前已实现基于 attempt 快照的选择题自动评分、总分、得分、正确题数、提交时间、成绩列表、成绩详情、题目明细、考生答案、正确答案、解析和附件展示。

## 5. 当前范围

当前仓库包含：

- 根文档、AI 协作规则、产品事实文档、计划模板、安全说明、贡献说明和开源协议。
- 后端 Spring Boot 工程，包含统一响应、错误码、全局异常处理、健康检查、Spring Security、JWT、Flyway migration、用户权限基础表、菜单表、角色菜单关系和管理员种子账号。
- 后端管理接口包含用户管理、用户 Excel、角色管理、部门管理、权限与菜单字典、题库管理、试题管理、试题 Excel、考试管理和成绩详情；内部方法级授权仍使用权限 code。
- 后端考试端接口包含考试任务、开始作答、提交试卷和我的成绩，并要求登录和考试权限。
- 前端 Vue 3 TypeScript 工程，包含登录页、首次改密页、统一控制台布局、顶部面包屑、已访问页面标签、四个一级菜单、路由守卫、API client、token store、用户管理、角色管理、部门管理、题库管理、试题管理、考试管理、考试中心、准备考试、在线作答、我的成绩、成绩详情、题目附件展示、403 和 404。系统管理显性子菜单只保留用户管理、角色管理和部门管理；考试管理显性子菜单只保留题库管理、试题管理和考试管理。
- Docker Compose 配置，描述 MySQL、Redis、后端和前端服务；本地 MySQL 使用 `13306`，Redis 使用 `16379`，避免和本机默认端口冲突。
- 根常规验证脚本，串联 Python 脚本编译、后端测试、前端类型检查、前端单元测试、前端构建、Docker Compose 配置检查和文档扫描，并严格处理命令失败。
- 根浏览器验收脚本，使用真实 Docker MySQL/Redis、真实 Spring Boot 后端、真实 Vue 前端和 Chromium，覆盖登录、控制台主菜单、顶部页面标签、管理端导航、角色保存、部门编码自动生成、用户导入导出、试题题库筛选、考试显示方式配置、准备考试、在线作答题型显示、本地音频/图片附件渲染、提交试卷和成绩复盘。

## 6. 验收标准

- 根目录存在 `README.md`、`AGENTS.md`、`spec.md`、`plan.example.md`、`LICENSE`、`SECURITY.md`、`CONTRIBUTING.md` 和 `.gitignore`。
- `.codex/skills/` 中存在开发、UI 和 plan 三个项目级 skill。
- 文档统一使用 kaoshi 作为项目名。
- 文档只记录当前确认的产品方向和工程规则。
- `backend/` 存在可测试的 Spring Boot 工程。
- `frontend/` 存在可类型检查和可构建的 Vue 3 TypeScript 工程。
- 管理员可以通过管理端维护用户、角色、权限关系和菜单关系，并通过 Excel 导入导出用户。
- 管理员可以通过管理端在线维护部门树。
- 管理员可以通过管理端维护题库、试题和考试，保存考试草稿并在发布时按题库规则生成发布快照。
- 管理员可以通过 Excel 模板导入和导出试题。
- 登录用户可以通过在线考试查看考试任务、准备考试、开始作答、提交试卷并查看成绩详情。
- 在线考试可以展示题目图片、音频和文件链接附件。
- 系统可以基于作答快照对单选题和多选题自动评分，并归档成绩。
- 未登录请求不能访问管理接口。
- `start_test.py` 是当前常规验证入口，不打开浏览器。
- `start_browser_test.py` 是当前真实浏览器验收入口，会重置本项目 Docker 数据库并运行 Chromium 自动验收。
- `stop_test.py` 可以停止测试相关服务。
