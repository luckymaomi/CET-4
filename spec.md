# gaokao Spec

`spec.md` 是 gaokao 的当前产品事实主干。它记录已经确认的产品定位、业务边界、核心概念、用户路径、技术方向和验收标准。

## 1. 产品定位

gaokao 是一个现代化考试与考试管理平台。

平台服务于学校、机构、团队和个人组织考试的完整流程：建设题库、维护试题、编排试卷、发布考试、考生作答、自动评分、成绩归档和结果查看。

## 2. 当前技术方向

- 后端使用 Java 21 和 Spring Boot 3。
- 数据库使用 MySQL 8。
- ORM 使用 MyBatis-Plus。
- 认证权限使用 Spring Security 和 JWT。
- 缓存、限流和考试运行状态辅助能力使用 Redis。
- 管理端使用 Vue 3、TypeScript、Vite 和 Element Plus。
- 考试端使用 Vue 3 和 TypeScript。
- 部署使用 Docker Compose 组织 MySQL、Redis、后端服务和前端服务。

## 3. 核心用户

- 系统管理员：管理平台配置、用户、角色、权限和全局数据。
- 考务管理员：维护题库、试题、试卷、考试安排和成绩。
- 教师或命题人：创建题目、维护解析、组卷和查看考试结果。
- 考生：查看考试任务、进入考试、作答、提交和查看结果。

## 4. 核心概念

### 4.1 用户与权限

用户是平台身份主体。角色和权限控制管理端能力、考试发布能力、题库维护能力和成绩查看范围。

### 4.2 题库

题库是试题的组织容器。题库支持分类、标签、题型、难度、知识点和归属范围。

### 4.3 试题

试题是考试内容的基本单位。试题包含题干、题型、选项、答案、解析、分值、难度、标签和附件。

### 4.4 试卷

试卷由试题组成。试卷记录题目顺序、分值快照、总分、限时、说明和组卷规则。

### 4.5 考试

考试引用试卷并定义可见范围、开始时间、结束时间、限时、提交规则、成绩规则和发布状态。

### 4.6 作答

作答记录考生每道题的答案、用时、提交状态、评分结果和最终成绩。

### 4.7 成绩

成绩是作答提交后的结果归档，包含总分、得分、正确率、用时、题目明细和可查看范围。
