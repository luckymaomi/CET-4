# Contributing

Thank you for your interest in contributing.

## How to Contribute

1. Open an issue or discussion for substantial changes before starting work.
2. Keep each change focused and easy to review.
3. Include tests or verification steps for behavior changes.
4. Update documentation when behavior, configuration, or public usage changes.
5. Make sure the project builds and the relevant checks pass before submitting.

## Pull Requests

- Use a clear title and description.
- Explain what changed and why.
- Link related issues when applicable.
- Keep unrelated refactors out of the same pull request.
- Be prepared to revise the change after review.

## Code Quality

- Prefer simple, maintainable solutions.
- Follow the existing style of the codebase.
- Avoid unnecessary dependencies.
- Handle errors explicitly.
- Do not commit generated files, local configuration, secrets, or temporary artifacts unless they are intentionally part of the project.

## Conduct

Be respectful and constructive. Focus discussion on the work, the evidence, and the desired outcome.
